# Petodo 项目协作规则

## 每次开始任务前

- 先阅读 `README.md`、`PROJECT_PROGRESS.md`、`NEXT_TASKS.md`。
- 先确认当前任务属于哪个模块，再决定改哪些文件。
- 如果发现 README、进度表、任务清单和实际代码不一致，先以实际代码为准，并在完成任务后更新进度文档。

## 每次修改代码前

- 明确本次只处理一个具体模块，不要一次性重构整个项目。
- 优先保证课程大作业能展示，先完成可运行、可演示的主流程。
- 不随意改无关文件，不回退已有改动。
- 遇到已有改动时，先判断是否和本次任务有关；无关则保留。

## 每次完成任务后

- 必须更新 `PROJECT_PROGRESS.md`，记录对应模块的最新状态。
- 必须更新 `NEXT_TASKS.md`，调整下一步任务优先级。
- 尽量先实际运行或检查结果，确认功能可用后再汇报。

## 每次回复用户时

必须主动说明：

1. 这次完成了什么
2. 项目进度有没有变化
3. 当前最重要的问题是什么
4. 下一步建议做什么
5. 哪些功能已经完成，哪些还没完成

## 当前项目优先级

- 第一优先级：保证主窗口、桌宠窗口、番茄钟、桌宠状态切换能稳定展示。
- 第二优先级：补齐喂食、饥饿值、本地保存等能增强展示效果的功能。
- 第三优先级：积分、商店、统计、隐藏、缩放等加分功能。
